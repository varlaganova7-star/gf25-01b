# Конспект: Системы управления базами данных (СУБД)

## 1. Понятие и архитектура СУБД

**СУБД** - комплекс программных и языковых средств для создания, ведения и управления базами данных.

### Трехуровневая архитектура ANSI/SPARC:
- **Внутренний уровень** - физическое хранение данных
- **Концептуальный уровень** - логическая структура БД
- **Внешний уровень** - представления для пользователей

**Преимущество**: обеспечение независимости данных (логической и физической)

## 2. Основные функции СУБД

###  Управление данными
- Создание и модификация структуры (DDL - `CREATE`, `ALTER`)
- Манипуляция данными (DML - `INSERT`, `SELECT`, `UPDATE`, `DELETE`)
- Поиск и фильтрация через индексы

###  Управление транзакциями
**Свойства ACID:**
- **Atomicity** - атомарность
- **Consistency** - согласованность  
- **Isolation** - изолированность
- **Durability** - долговечность

###  Управление хранением
- Управление дисковым пространством
- Методы доступа (индексы, хеширование)
- Кэширование в оперативной памяти
- Оптимизация ввода-вывода

###  Поддержка языков
**SQL включает:**
- DDL - описание данных
- DML - манипулирование данными
- DCL - управление доступом
- TCL - управление транзакциями

###  Безопасность
- **Аутентификация** - проверка подлинности
- **Авторизация** - права доступа
- **Аудит** - регистрация действий

## 3. Ключевые возможности

###  Многопользовательский доступ
- Одновременная работа многих пользователей
- Механизмы управления параллелизмом
- Предотвращение конфликтов

###  Резервное копирование
- Создание бэкапов
- Восстановление после сбоев
- Обеспечение непрерывности бизнеса

###  Целостность данных
**Обеспечивается через:**
- Ограничения целостности:
  - Первичные ключи (`PRIMARY KEY`)
  - Внешние ключи (`FOREIGN KEY`) 
  - Уникальность (`UNIQUE`)
  - Проверочные ограничения (`CHECK`)
- Триггеры

###  Администрирование
- Графические консоли управления
- Мониторинг производительности
- Тонкая настройка (тюнинг)

###  Поддержка моделей данных
- **Реляционные СУБД** - традиционные
- **NoSQL**:
  - Документоориентированные (MongoDB)
  - Графовые (Neo4j)
