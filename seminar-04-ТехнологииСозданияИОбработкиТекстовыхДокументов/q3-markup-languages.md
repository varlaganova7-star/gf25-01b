# Языки разметки

### Что это такое и зачем?

> Язык разметки — это набор меток, которые вставляют прямо в текст. Они показывают: вот заголовок, вот абзац, вот список, вот картинка. Без них текст — просто строка символов. С ними — документ, который компьютер понимает и может быстро превратить в сайт, PDF, печатный лист или озвучку. Разметка проверяет себя сама: не хватает тега — ошибка. Поисковики видят структуру и поднимают страницу выше. Слабовидящие слушают через программы — всё логично. Главный плюс: содержание в одном месте, внешний вид — в другом. Один файл — и он выглядит по-разному на телефоне, мониторе или бумаге. Ничего не переделываешь.


### Как всё началось?

- **1960-е, IBM**: Чарльз Голдфарб и команда создали GML — первые метки, чтобы переносить документы между машинами без потерь.
- **1986**: SGML — международный стандарт, как конструктор: из него делали свои разметки для книг, инструкций, науки.
- **1998**: XML — проще, жёстче, для обмена данными между программами.
- **HTML**: 1995 — 2.0, потом 3.2 и 4.01 (много команд на стиль). 2000 — XHTML (под XML, всё строго). 2014 — HTML5: не только вид, но и смысл (где шапка, где статья, где меню), плюс видео, формы, адаптация под экраны.
- **В России**: с 90-х — в вузах, библиотеках, на заводах, в налоговой. SGML и XML — чтобы отчёты не ломались при переносе.


### Какие бывают?

1. **Стилевые** (старый HTML): <font color="red"\>, <b\>, <center\> — красиво, но компьютер не понимает, что важно.
2. **Смысловые** (HTML5, XML): <strong\>, <em\>, <article\>, <nav\> — машина сразу видит: это заголовок, это меню, это цитата.
3. **Узкие**
    - MathML — только формулы (E = mc²).
    - SVG — векторная графика
    - MusicXML — ноты.
    - LaTeX — научные тексты: формулы, ссылки, таблицы, авто-нумерация.
4. **Легкие**: Markdown — # Заголовок, - пункт, `код`, > цитата. Читается как есть, конвертируется во всё подряд.


## Роль языков разметки в структурировании документов

Они превращают хаос текста в чёткую иерархию, понятную и человеку, и машине. Конкретно:

1. 	**Метки-ориентиры** - каждый кусок получает имя: <h1\> — главный заголовок, <p\> — абзац, <ul\> — список. Компьютер за секунды находит нужное, строит оглавление, выделяет главное.
2. **Вложенность** - как папки: раздел → подраздел → текст + формула + рисунок. Легко копировать, править, конвертировать — структура не рушится.
3. **Содержание ≠ оформление** — текст в файле, стили отдельно. Один документ → сайт, PDF, печать, ePub — без переписывания.
4. **Метаданные** — автор, дата, ключевые слова, язык. Удобно искать, фильтровать, проверять права.

**Итог:** разметка делает документ надёжным, гибким и вечным — он работает сегодня и через 30 лет, с любым софтом.

### Примеры:
- **HTML5:**  
<header\><h3\>Заголовок</h3\></header\>
<nav\>Главная | О нас</nav\>
<article\><h3\>Часть 1</h3\><p\>Текст...</p\></article\>   
> Этот код будет выглядеть так:
---
<header><h3>Заголовок</h3></header>
<nav>Главная | О нас</nav>
<article><h3>Часть 1</h3><p>Текст...</p></article>

Поисковики видят структуру, скрин-ридеры читают по порядку.

- **XML:** в банках — каждая строка проверяется. Нет закрывающего тега — документ не пройдёт.
- **LaTeX:**  
  \section{Введение}
  $E = mc^2$  
Всё нумеруется само: разделы, рисунки, ссылки.

### Плюсы:
- Машина читает миллионы строк за секунды.
- Один текст — куча форматов.
- Меньше ошибок, проще учить.
- Доступно всем: ИИ, архивы, слабовидящие.
- Документ живёт десятилетиями.

### Минусы:
- Нет универсального стандарта — для каждой задачи своя схема.
- Многие пишут по-старому: <div\> вместо <section\>.
- Нужны программы: Overleaf, Pandoc, валидаторы.
- Новичкам сложно: теги, отступы, закрытие.
- Результат не всегда виден сразу.

### Что выбрать?
- **Веб** — HTML5
- **Наука** — LaTeX
- **Обмен данными** — XML
- **Команды, заметки** — Markdown  

**Вывод:** разметка — это скелет документа. Она организует порядок, который работает везде и всегда. Особенно важно, когда текст читают не только люди, но и машины.
