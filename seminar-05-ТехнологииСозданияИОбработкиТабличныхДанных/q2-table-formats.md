# Основные форматы хранения и обмена табличной информацией

> Сложно представить себе вид человеческой деятельности, где нет таблиц. И это неудивительно, ведь такая структура данных широко используется в процессе решения задач интеллектуальной и управленческой деятельности человека. 

Эта структура данных широко используется в различных отраслях:

- **Наука и техника** (тригонометрия, логарифмы, таблицы функций)
- **Инженерные расчеты** (табличное представление механических свойств разных материалов)
- **Финансы** (курсы акций, кросс-курсы валют)
- **Бухгалтерский учет** (баланс предприятия, оборотные ведомости)
- **Документация АХО** (административно-хозяйственного отдела)

## Электронные таблицы

Если представить таблицы в электронной форме, задействовав компьютер, мы получим возможность как отображения, так и обработки данных. Именно для этого и предназначены электронные таблицы.

>**Электронная таблица** - инструмент, представляющий собой сетку из ячеек для хранения данных, используемый для организации, анализа и обработки данных.

### Основные функции

- **Хранение данных** - хранение больших объемов данных в удобной форме
- **Расчеты и формулы** - автоматические вычисления с помощью формул
- **Создание отчетов** - создание отчетов и документации на основе данных

>**Табличные процессоры** — это программы для создания и обработки электронных таблиц, которые позволяют вводить, редактировать и анализировать данные в виде двумерных массивов (строк и столбцов).

### Популярные табличные процессоры:

| Программа | Описание |
|-----------|----------|
| **Microsoft Excel** | Мощный табличный процессор, часть пакета Microsoft Office. Предлагает расширенные возможности для анализа данных, визуализации, построения диаграмм и сложных вычислений. Поддерживает макросы на VBA, интеграцию с другими продуктами Microsoft.|

| **LibreOffice Calc** | Мощный табличный процессор из свободного офисного пакета LibreOffice. Полная совместимость с форматами Microsoft Excel, поддержка макросов и расширенных функций. |
| **KSpread** | Табличный процессор с открытым исходным кодом, он входил в состав офисного пакета KOffice и предоставляет инструменты для работы с электронными таблицами, сложными вычислениями и визуализацией данных. Когда речь идет о хранении и обработке больших данных, для решения информационных задач уже используют БД и СУБД.

## Базы данных и СУБД

### Что такое база данных

> **База данных** - ээто организованная коллекция данных, которая хранится и обрабатывается с помощью компьютерной системы. В бизнесе базы данных используются для хранения информации о клиентах, сотрудниках, документах и прочем. Также базы данных способны анализировать и сортировать информацию.

> **Система управления базами данных (СУБД)** - программное обеспечение для создания и управления базами данных, обеспечивающее удобный доступ к данным, безопасность, контроль целостности данных, поддерживают различные операции с данными, такие как добавление, удаление, обновление, поиск и т.д.

## Сравнение баз данных и электронных таблиц

| Критерий | Базы данных | Электронные таблицы |
|----------|-------------|---------------------|
| **Масштабируемость** | Базы данных способны обрабатывать большие объемы данных и обеспечивать быстрый доступ к ним | Электронные таблицы могут быть менее эффективными для работы с большими объемами данных из-за ограничений по производительности и масштабируемости |
| **Безопасность** | Высокий уровень, шифрование, контроль доступа | Электронные таблицы имеют более ограниченные возможности по контролю доступа к данным и защите информации. |
| **Специализация** | Базы данных специализированы для работы с большими объемами данных, поддерживают сложные запросы и аналитику данных | Электронные таблицы предназначены для более простых операций с данными, таких как расчеты, отчеты на небольшом объеме информации |

## Форматы хранения и обмена 

#### XLSX (Microsoft Excel Open XML Spreadsheet)
- Широко поддерживается, множество программ и сервисов могут его открывать.
- Поддержка форматов изображений, формул, графиков высока.
- Достаточно компактный размер файлов при хорошем качестве.

`Рекомендация: Идеально подходит для большинства задач, совместной работы и обмена файлами.`

#### ODS (OpenDocument Spreadsheet)
- Стандартный открытый формат, поддерживается многими программами.
- Довольно компактный размер файлов.
- Менее распространенный, чем XLSX, немного меньше функций, доступных в нём.

`Рекомендация: Хороший выбор для тех, кто предпочитает открытые форматы.`

#### CSV (Comma Separated Values)
- Простой текстовый формат, легко импортируется многими программами.
- Лучше подходит для обмена данными и их хранения, чем для сложных расчетов и визуализации.

`Рекомендация: Используйте для структурированных данных, которые не требуют сложной обработки или визуализации в Excel.`

#### XLS (Microsoft Excel Binary Spreadsheet)
- Старый, несовременный формат, который может вызвать проблемы с совместимостью.
- Встречаются ограничения поддержки в современных программах.

`Рекомендация: Избегайте, если есть альтернатива.`

#### Parquet
-  Создан специально для хранения табличных данных.
- Основное применение: высокопроизводительная аналитика, хранение больших датасетов.
- Ключевое отличие от CSV в том, что Parquet поддерживает сложные типы данных: массивы (array), структуры (struct), и карты (map). 

`Рекомендации: следует использовать для хранения и обработки больших объемов структурированных данных, где критически важны высокая производительность запросов, эффективное сжатие и возможность чтения только необходимых столбцов.`
